# Спецификация требований

## 1. Введение

### 1.1 Цель
Создать веб-платформу для распространения игры "The Virus Cult" и взаимодействия с сообществом.

### 1.2 Целевая аудитория
- Игроки, желающие скачать игру
- Фанаты, желающие оставить отзыв
- Желающие поддержать проект

## 2. Функциональные требования

### FR-01: Регистрация пользователя
| Элемент | Описание |
|---------|----------|
| Приоритет | High |
| Предусловие | Email не зарегистрирован |
| Постусловие | Создан пользователь, отправлено письмо |
| Исключение | Email уже существует → ошибка 422 |

**Входные данные:**
- `name` (string, required, max:255)
- `email` (string, required, email, unique)
- `password` (string, required, min:8, confirmed)
- `newsletter` (boolean, optional)

### FR-02: Верификация email
| Элемент | Описание |
|---------|----------|
| Приоритет | High |
| Предусловие | Пользователь зарегистрирован |
| Постусловие | email_verified_at установлен |

### FR-03: Авторизация
| Элемент | Описание |
|---------|----------|
| Приоритет | High |
| Вход | email + password |
| Выход | POST /logout |

### FR-04: Отзывы
| Элемент | Описание |
|---------|----------|
| Приоритет | Medium |
| Ограничения | 1 отзыв на пользователя |
| Рейтинг | 1-5 звезд |

### FR-05: Скачивание игры
| Элемент | Описание |
|---------|----------|
| Приоритет | High |
| Платформы | Windows (.zip), Linux (.tar.bz2), Mac (.zip) |

### FR-06: Пожертвования
| Элемент | Описание |
|---------|----------|
| Приоритет | Low |
| Реализация | Редирект на DonationAlerts |

## 3. Нефункциональные требования

### NFR-01: Производительность
- Время ответа < 200ms (95% запросов)
- Поддержка до 100 одновременных пользователей

### NFR-02: Безопасность
- Хеширование паролей (bcrypt)
- CSRF защита
- Rate limiting (6 запросов/мин на верификацию)

### NFR-03: Доступность
- 99.5% uptime
- Graceful degradation при недоступности внешних сервисов

### NFR-04: Тестируемость
- Покрытие кода > 70%
- Все тесты проходят за < 5 минут
